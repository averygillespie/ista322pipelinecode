Spotify Popularity & Urban Air Quality Analysis
Project Overview

This project integrates Spotify track data with city-level air quality data to explore trends across major global cities. The objective was to design a structured data pipeline, clean and normalize multiple datasets, and build a relational database to support analytical queries and visualization.

Objectives

Integrate data from multiple external sources

Clean, transform, and normalize structured datasets

Design and query a relational database using SQL

Analyze cross-city trends in music popularity and environmental metrics

Build an interactive dashboard to support exploratory analysis

Data Sources

Spotify API (track metadata and popularity metrics)

City-level air quality datasets

Methods & Tools

Python for data extraction and processing

SQL for database design and querying

Data cleaning and transformation techniques

Streamlit for dashboard development

Data visualization and exploratory analysis

Key Features

Multi-source data integration

Relational database structure

Aggregated city-level comparisons

Popularity trend analysis

Interactive dashboard for user-driven exploration
